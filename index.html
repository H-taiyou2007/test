Quick Reference

Git & GitHub
チートシート

最初のセットアップから、うっかりの取り消し方まで。
コマンド右の「コピー」でそのまま貼り付けできます。

Step 1 — はじめの一歩

セットアップ

Gitを使い始めるときに、最初の一度だけ行う設定たち。

git init

今いるフォルダを、新しいGitリポジトリとして初期化する

git clone <URL>

すでにあるリポジトリを、自分のパソコンに複製してくる

git config --global user.name "名前"

コミットに記録される、あなたの名前を設定する

git config --global user.email "メール"

コミットに記録される、あなたのメールアドレスを設定する

Step 2 — 日々の記録

日常のワークフロー

ファイルを変更したあと、その変更を記録として残すための基本コマンド。

git status

今どのファイルが変更されているか、状態を確認する

git add .

変更したファイルすべてを、記録の対象に加える

git commit -m "メッセージ"

加えた変更に、ひとまとまりの記録(コミット)として名前をつけて保存する

git log --oneline

これまでのコミットの履歴を、1行ずつ簡潔に表示する

Step 3 — 枝分かれ

ブランチを操る

試したい変更を、メインの履歴から分けて作業するための仕組み。

git branch

今あるブランチの一覧を表示する

git switch -c <名前>

新しいブランチを作り、そのままそこへ移動する

git switch <名前>

指定したブランチへ移動する

git merge <名前>

指定したブランチの変更を、今いるブランチに取り込む

Step 4 — やり直し

元に戻す

うっかりミスをしても大丈夫。落ち着いて、状況に合った方法を選ぶ。

git restore <ファイル>

まだコミットしていない変更を、直前の状態に戻す

git reset --soft HEAD~1

直前のコミットを取り消す(変更内容はそのまま残る)

git revert <コミット>

過去のコミットを打ち消す、新しいコミットを作る(履歴を残したまま安全に戻す)

git stash

作業中の変更を、いったん脇にどかして保管しておく

Step 5 — つながる

GitHubと連携する

手元の記録を、GitHub上のリポジトリと行き来させる。

git remote add origin <URL>

手元のリポジトリに、GitHub上の場所を「origin」として登録する

git push -u origin main

手元の記録をGitHubへ送る(以後は git push だけでよくなる)

git pull

GitHub上の最新の変更を、手元に取り込む

git remote -v

今どこと連携しているか、登録済みの接続先を確認する
